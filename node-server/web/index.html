Useful bookmarks to elasticsearch FFF monitoring

  • F3Mon (Node.js version)
    Latest and greatest page with run information, stream rates, merging completion and microstate monitoring.

  • F3Mon (PHP version)
    Last PHP-powered version of the UI.

  • F3Mon (legacy version)
    Older PHP-powered version of F3Mon page

  • F3Mon (legacy version) with full detail microstate chart
    Warning: can overload browser if HLT menu has many modules

  • BigPic
    Monitoring of appliance resources and elasticsearch status

  • Ramdisk and Output rate from Kibana-cdaq (obsolete)
    Interval and refresh rate can be adjusted using the timepicker in the top right of the page.

  • Input rate from index file via Tribe (obsolete)
    The run must be selected by hand by going to the settings of the page (the gear in the top right corner), selecting the "Index" tab and putting the run number as "default index" (don't forget the leading "run" and the trailing "*"). If you don't select a run number (i.e. use run* as default index) be careful that the time span is not too wide or the browser will collapse.

Note: if you are loading any *directory* link through the cmsdaq0 proxy, make sure to add a trailing slash character to the URL, otherwise links will be broken (links from this page are correct).
